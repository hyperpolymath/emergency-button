== Philosophy

Emergency Button is designed for *immediate crises*. It's not a system administration tool—it's a fire extinguisher.

|===
|Emergency Button |Other Tools

|Panic mode |Calm, methodical
|One button |Many options
|Immediate relief |Long-term solutions
|Works offline |May need connectivity
|No configuration |Highly configurable
|Safe defaults |Powerful options
|===

== The Button

[source,bash]
----
emergency-button
----

That's it. One command. One button.

== Core Capabilities

=== Disk Emergency
* Identify space consumers
* Clear safe caches
* Report large/old files
* Prune containers

=== Memory Emergency
* Identify memory hogs
* Offer to kill safe targets
* Clear filesystem caches

=== Process Emergency
* Identify CPU hogs
* Kill/nice runaway processes

=== Network Emergency → handoff to complete-internet-repair
* Quick connectivity check
* Handoff to specialized tool

== Safety Guarantees

Emergency Button will *never*:

* Delete user data without confirmation
* Modify system configuration
* Require root for basic operations
* Make things worse

== Usage

[source,bash]
----
# Interactive (default)
emergency-button

# Headless (for scripts)
emergency-button --headless --safe-only

# Dry run
emergency-button --dry-run

# Maximum safe cleanup, minimal prompts
emergency-button --panic
----
